    Inside this new section, add `computeTodayWindow(now)` as the first helper. The function is PURE: no DOM access, no setState, no fetch.

    Algorithm verbatim from 03-RESEARCH.md Section 9:

    - Default `now` to `new Date()`.
    - Compute `isoDate` as `YYYY-MM-DD` using `getFullYear()`, `getMonth()+1`, `getDate()` zero-padded.
    - Compute `dayOfWeek = now.getDay()` (0 Sunday .. 6 Saturday).
    - Use a `weekdaysDe` array `['Sonntag', 'Montag', 'Dienstag', 'Mittwoch', 'Donnerstag', 'Freitag', 'Samstag']`.
    - Build `todayLabel = weekdaysDe[dayOfWeek] + ' ' + isoDate`.
    - Branch 1 (weekend): if `dayOfWeek === 0 || dayOfWeek === 6` return `{ open: false, todayLabel, windowText: 'Closed (weekend)', note: '' }`.
    - Branch 2 (closure entry): find `closure` in `MIGRATIONSAMT.closures2026` whose `date === isoDate`. If `closure.type === 'closed'` return `{ open: false, todayLabel, windowText: 'Closed (' + closure.name + ')', note: '' }`. If `closure.type === 'reduced'` use the morning window from `MIGRATIONSAMT.phoneHours.mondayToFriday[0]` and return `{ open: true, todayLabel, windowText: morning.from + '-' + closure.closesAt + ' (reduced hours, ' + closure.name + ')', note: '' }`.
    - Branch 3 (normal weekday): build `windowText` by joining `MIGRATIONSAMT.phoneHours.mondayToFriday` entries as `from + '-' + to` separated by `, `.
    - 24 Dec 2026 footnote: if `isoDate === '2026-12-24'` set `note = '(24 Dec schedule unconfirmed; verify on zh.ch before calling)'`. Otherwise empty string.
    - Return `{ open: true, todayLabel, windowText, note }`.

    Use a regular ASCII hyphen `-` for the time-range separator. Use a regular comma + space for joining windows.

    Hard constraints:
    - The function calls `new Date()` at most once (via the parameter default).
    - No DOM access. No console.log of dates.
    - String concatenation uses simple `+` (beginner-friendly per CLAUDE.md).
    - One paragraph of comment above the function listing the four edge cases handled (weekend / closed / reduced / 24 Dec footnote).

    Also add an inline doc-comment block immediately under the function (NOT executed) showing the expected outputs for these reference dates:
